🔥 三条最佳经验
1免费模型调研第一步：先查清哪些真的免费，旧推荐不可靠
为 Hermes Agent 选免费主模型时发现残酷事实：上一轮推荐的 gpt-oss-120b:free、Groq 全系、deepseek-r1:free 全部失效（已下架或已付费）。OpenRouter 免费层每月轮换。实测 2026-07-22 在线 10 款免费模型，最终产出完整参考卡。
→ 可复用：做免费模型选型前，用 OpenRouter API + freellm.net + 社区反馈三重验证，不盲目信旧推荐
2config 被自动改写时，重新推断实际配置比"纠正"更明智
Hermes 运行时自动改写 config.yaml（base_url/provider/default 全变）。多次手动改回又被改写。最后发现实际生效的是 OpenRouter 端的 nemotron-3-ultra:free，而非我配的 NVIDIA 直连 gpt-oss-120b。
→ 可复用：配置被改写后，先查 runtime 实际用什么在跑，推断变更来源，再决策是否干预
3白天首要做事，其余放松——自动化提醒同步修正
主人今天明确了"白天首要做事，其余时间放松练霎哈嘉/连接"。相应调整了 3 个自动化提醒内容，保持一致。
→ 可复用：自动化提醒的内容要和主人的当下态度保持一致，避免脱节
⚠️ 做得不好的事
1Hermes config 被自动改写后盲目重配
在 config.yaml 上花了大量不必要时间：被改写→读出来→改回去→又被改写。第一次发现改写后应推断变更来源（用户操作/程序自改写），而非假设冲突就覆盖。
→ 改进：先查 runtime 实际谁在跑、配置为何变了，再决策
